# Linux node installer package (headless)

**What auditors should read first.** This directory is the **only** place that defines the scripts shipped inside:

- `bloodstone-node-*-linux-x86_64.tar.gz`
- `bloodstone-node-*-linux-aarch64.tar.gz` (Raspberry Pi / ARM64)

Everything else in the monorepo (portal, Android miner, stratum, fleet, etc.) is **not** in the tarball unless explicitly listed below.

## Shipped files (tarball contents)

| File in package | Source in monorepo | Role |
|-----------------|--------------------|------|
| `bin/bloodstoned` | Built from `core/` | Full node daemon |
| `bin/bloodstone-cli` | Built from `core/` | RPC CLI |
| `start-node.sh` | **`packages/linux-node/start-node.sh`** | First-run conf + bootstrap + start |
| `install-chain-bootstrap.sh` | **`packages/linux-node/install-chain-bootstrap.sh`** | Tip snapshot download/verify/extract |
| `bloodstone.conf.example` | **`packages/linux-node/bloodstone.conf.example`** | Template conf (seeds, RPC bind) |
| `README.txt` | Generated by packager | Human quick start |

## Not in this package

Do **not** treat these as part of the Linux node installer audit surface:

- `ops/*` pool/stratum/watchdog scripts (VPS-only; many files, unrelated to the tarball)
- `miner-android/`, `miner-web/`, `portal/`, `pi-fleet`
- `release-downloads` branch binary dump (artifacts only)

## How the tarball is built

| Arch | Packager script (VPS) | Output |
|------|----------------------|--------|
| ARM64 / Pi | `/root/build-bloodstone-node-arm64-package.sh` | `bloodstone-node-VERSION-linux-aarch64.tar.gz` |
| x86_64 | `/root/build-bloodstone-node-distribution.sh` | `bloodstone-node-VERSION-linux-x86_64.tar.gz` |

Both copy scripts **from this directory** into the staging tree.

## Security / audit notes

See: `docs/Bloodstone-Node-0.7.6-Security-Audit-Remediation.md`  
High-severity fixes live in `start-node.sh` (conf mode 600) and `install-chain-bootstrap.sh` (safe tar + HTTPS curl).

### Operator env (installer-related only)

| Variable | Effect |
|----------|--------|
| `BLOODSTONE_DATADIR` | Data directory (default `~/.bloodstone`) |
| `BLOODSTONE_SEEDS` / `BLOODSTONE_SEED_NODES` | Override default seed peers |
| `BLOODSTONE_SKIP_BOOTSTRAP` | Skip tip snapshot (full IBD) |
| `BLOODSTONE_FORCE_BOOTSTRAP` | Re-download tip snapshot |
| `BLOODSTONE_PUBLIC_ROOT` | Base URL for bootstrap download |
| `BLOODSTONE_CHAIN_BOOTSTRAP_URL` | Full bootstrap tarball URL |
